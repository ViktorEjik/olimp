# Олимпиада — генераторы тестов

Репозиторий содержит условия задач (LaTeX) и эталонные решения с генераторами входных данных. Каждая задача оценивается в **100 баллов**: **4 группы** тестов по **25 баллов**, сложность растёт от группы 1 к группе 4.

## Требования

- Python 3.10+ (достаточно стандартной библиотеки)

## Структура задачи

```
task_N/
  taskN.tex       # условие
  solution.py     # эталонное решение (stdin → stdout)
  gen.py          # генератор тестов
  tests/          # создаётся локально, не коммитится
    group1/
      test01.in
      test01.ans
      ...
    group2/
    ...
```

Общие утилиты: [tools/gen_common.py](tools/gen_common.py).

## Генерация тестов

Из корня репозитория:

```bash
python task_1/gen.py --group 1 --out task_1/tests
```

### Параметры

| Параметр | Описание |
|----------|----------|
| `--group` | Номер группы: `1`, `2`, `3` или `4` |
| `--out` | Каталог для вывода (обычно `task_N/tests`) |
| `--seed` | Seed для `random` (по умолчанию `1000 + номер_группы`) |

Примеры:

```bash
# одна группа
python task_3/gen.py --group 2 --out task_3/tests

# воспроизводимый набор
python task_5/gen.py --group 4 --out task_5/tests --seed 42

# все группы одной задачи
for g in 1 2 3 4; do
  python task_2/gen.py --group "$g" --out task_2/tests
done
```

Все задачи сразу:

```bash
for t in 1 2 3 4 5; do
  for g in 1 2 3 4; do
    python "task_${t}/gen.py" --group "$g" --out "task_${t}/tests"
  done
done
```

### Результат

- В группе **10 тестов**: `test01.in` … `test10.in`
- Ответы `testNN.ans` строятся автоматически запуском `solution.py` на соответствующем `.in`
- `test01` в **group1** — официальный пример из условия (где он есть)

Каталог `tests/` указан в `.gitignore` и **не попадает в git** — его нужно сгенерировать локально на каждой машине.

## Проверка вручную

Запуск решения на одном тесте:

```bash
python task_3/solution.py < task_3/tests/group1/test01.in
diff -u task_3/tests/group1/test01.ans <(python task_3/solution.py < task_3/tests/group1/test01.in)
```

Сверка всей группы:

```bash
g=1
for f in task_3/tests/group${g}/*.in; do
  base="${f%.in}"
  python task_3/solution.py < "$f" | diff -u "${base}.ans" -
done && echo "group $g OK"
```

## Группы сложности

Подробные лимиты и идеи тестов — в шапке каждого `gen.py`. Кратко:

| Задача | Группа 1 | Группа 4 (максимум по условию) |
|--------|----------|--------------------------------|
| 1. Супердроби | малые числа | полный диапазон int32 |
| 2. Квадродерево | мелкие деревья | длина строки до 1365 |
| 3. Лифт | мало запросов | N ≤ 999, \|A\|, \|B\|, \|X\| < 1000 |
| 4. Гномьи сундуки | N ≤ 5 | N = 500, длина записи до 100 |
| 5. ФБЦ-число | N ≤ 20 | N = 50000 |

## Сборка условий (PDF)

```bash
make
```

Результат: `main.pdf`.

## Замечания

- **Задача 1:** для примера `1 2 1 3 1 4` эталон даёт `9 1`, а не `13 4` из PDF — перебор всех перестановок подтверждает `9/1` как максимум.
- При ошибке генератора смотрите traceback: чаще всего падает `solution.py` на некорректном входе, сгенерированном в `gen.py`.

## Добавление новой задачи

1. Создать `task_N/solution.py` (читает stdin, пишет stdout).
2. Создать `task_N/gen.py` с функцией `cases_for_group(group, seed) -> list[str]` и вызовом `write_tests` из `gen_common`.
3. В `group1` первым тестом включить пример из условия.
4. Проверить: `python task_N/gen.py --group 1 --out task_N/tests`.
